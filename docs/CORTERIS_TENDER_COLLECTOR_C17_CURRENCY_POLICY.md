# Corteris Tender Collector C17

## Единая валютная политика

C17 делает валюту частью денежного контракта поиска, фильтрации и рейтинга.
Суммы разных валют больше не сравниваются как будто они выражены в рублях.

## Нормализация

`TenderMoney` хранит канонический трёхбуквенный код ISO 4217 в верхнем
регистре. Поддерживаются распространённые входные обозначения:

- `RUR`, `руб.`, `₽`, `643` → `RUB`;
- `$`, `доллар США`, `840` → `USD`;
- `€`, `евро`, `978` → `EUR`;
- `юань`, `156` → `CNY`.

Неизвестные текстовые обозначения и некорректные коды отклоняются, чтобы
ошибка источника не превращалась в неверную финансовую оценку.

## Поиск и профили

`TenderSearchQuery`, `TenderFilterOptions` и `TenderSearchProfile` содержат
`price_currency`. Значение по умолчанию — `RUB`, поэтому старые профили и
вызовы API остаются совместимыми. Валюта сохраняется в JSON-профилях и
checkpoint fingerprint.

ЕИС и Портал поставщиков Москвы применяют серверные границы цены только для
`RUB`, поскольку их текущие адаптеры не подтверждают поддержку валютной
конвертации. Для другой валюты рублёвые параметры в запрос не отправляются.

## Фильтрация и рейтинг

Если валюта тендера отличается от валюты заданного диапазона:

- локальный фильтр помечает несовпадение валют вместо сравнения чисел;
- рейтинг участия не применяет рублёвые пороги и требует ручного курса;
- автоматическая конвертация не выполняется.

Такой подход сохраняет воспроизводимость: курс, дата курса и источник курса
должны появиться отдельным проверяемым этапом, прежде чем система сможет
пересчитывать валютные суммы автоматически.
