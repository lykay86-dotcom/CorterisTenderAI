# Collector C4: нативный асинхронный провайдер ЕИС

## Статус интеграции

Провайдер работает в режиме `public_html_async` и использует публичные,
неавторизованные страницы ЕИС:

- публичный расширенный поиск закупок;
- публичную карточку закупки, найденную через поиск по номеру;
- публичную вкладку документов;
- публичную главную страницу для проверки состояния.

`public_api=False`. Коммит не выдаёт HTML-парсер за официальный API и не
использует вымышленные XML/JSON endpoints.

## Схема

```text
AsyncEisTenderProvider
    -> AsyncHttpClient (httpx.AsyncClient)
    -> per-domain RateLimiter
    -> SSL / timeout / 429 / 5xx retry
    -> EisHtmlParser (общий для sync и async)
    -> UnifiedTender
    -> AsyncProviderSearchEngine
    -> CollectorService
    -> tender_registry.sqlite3
```

Старый синхронный `EisTenderProvider` сохранён для обратной совместимости.
Общие функции построения URL, фильтрации и вкладки документов вынесены в
`app/tenders/providers/eis.py` и используются обоими адаптерами.

## Инкрементальный режим

Публичная HTML-страница не предоставляет подтверждённый документированный
cursor изменений. Поэтому checkpoint хранит время последнего успешного
запроса и применяет скользящее окно даты публикации (по умолчанию 14 дней).

Это позволяет уменьшить повторную выборку, но не гарантирует обнаружение
изменения старой закупки за пределами окна. Полноценный update feed должен
быть добавлен позже только после проверки официального сервиса/выгрузки.

Checkpoint сохраняет:

- scope поискового профиля;
- watermark;
- страницу и next page token;
- количество найденных записей;
- параметры фактического запроса;
- стратегию и размер overlap-окна.

## Ограничения

- CAPTCHA и HTTP 403 не обходятся.
- Авторизованные разделы не используются.
- Протоколы и разъяснения пока не имеют отдельной типизированной модели;
  доступные публичные файлы извлекаются со вкладки документов.
- Live-проверка зависит от доступности ЕИС, DNS, VPN, прокси и антивирусной
  HTTPS-инспекции на компьютере пользователя.
- Во время сборки внешние запросы не выполнялись; тесты используют сохранённые
  HTML fixtures и `httpx.MockTransport`.

## Ручная проверка

```powershell
.\.venv\Scripts\python.exe scripts\check_eis_async.py
```

Тестовый поиск:

```powershell
.\.venv\Scripts\python.exe scripts\check_eis_async.py `
    --search "видеонаблюдение"
```

Скрипт выводит реальный статус и не заменяет ошибку фиктивным успехом.
