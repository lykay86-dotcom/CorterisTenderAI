# Collector C6 — каталог коммерческих площадок

## Назначение

C6 добавляет восемь расширяемых адаптеров коммерческих площадок, но не
имитирует рабочее подключение. Для каждой площадки фиксируются отдельные
настройки, секреты и состояние готовности. Сетевые методы до подтверждения
реального разрешённого API-контракта возвращают `ProviderNotConfiguredError`.

## Площадки

- B2B-Center;
- ЭТП ГПБ;
- Фабрикант;
- ТЭК-Торг;
- OTC;
- Сбер А — коммерческие закупки;
- РТС-тендер — коммерческие закупки;
- Росэлторг — коммерческие закупки.

## Состояния

- `disabled` — источник отключён пользователем;
- `contract_required` — не подтверждены договор и разрешённый способ доступа;
- `credentials_required` — требуется ключ или учётные данные;
- `endpoint_required` — нужен подтверждённый адрес API;
- `ready_for_verification` — настройки заполнены, но реальный ответ ещё не
  проверен и источник не считается рабочим.

Состояния `available` или `working` для коммерческих источников C6 не
выставляются.

## Хранение настроек

Не секретные параметры сохраняются в пользовательском каталоге данных:

```text
commercial_providers.json
```

В JSON разрешены только:

- `enabled`;
- `access_confirmed`;
- `api_base_url`.

API-ключи читаются сначала из переменной окружения, затем из Windows keyring.
Они не записываются в JSON, диагностические отчёты или `repr` объектов.

## Переменные окружения

Для каждой площадки используется единая схема:

```text
CORTERIS_<PREFIX>_ENABLED=true
CORTERIS_<PREFIX>_ACCESS_CONFIRMED=true
CORTERIS_<PREFIX>_API_KEY=...
CORTERIS_<PREFIX>_API_BASE_URL=https://...
```

Префиксы:

```text
B2B
GPB
FABRIKANT
TEK_TORG
OTC
SBER_COMMERCIAL
RTS_COMMERCIAL
ROSELTORG_COMMERCIAL
```

Значение `ACCESS_CONFIRMED=true` следует задавать только после проверки права
использовать API или иную структурированную выгрузку площадки.

## Диагностика

Команда не выполняет сетевых запросов:

```powershell
.\.venv\Scripts\python.exe scripts\check_commercial_providers.py
```

JSON-вывод без секретов:

```powershell
.\.venv\Scripts\python.exe scripts\check_commercial_providers.py --json
```

## Подключение к поисковому движку

По умолчанию коммерческие адаптеры не добавляются в рабочий поиск. Явное
включение доступно только для будущего интерфейса и диагностики:

```python
providers = create_default_async_providers(
    network_runtime,
    include_commercial_catalog=True,
    commercial_catalog=catalog,
)
```

В движок попадут только провайдеры с `enabled=True`. До реализации конкретного
API они честно вернут результат `not_configured`, не остановив остальные
источники.

## Следующий шаг для конкретной площадки

1. Получить договор, документацию или подтверждение разрешённой выгрузки.
2. Настроить endpoint и секрет через окружение/keyring.
3. Выполнить контролируемый запрос диагностическим скриптом.
4. Сохранить обезличенный реальный fixture ответа.
5. Зафиксировать карту полей и ограничения частоты.
6. Реализовать отдельный нативный `AsyncTenderProvider`.
7. Только после успешных интеграционных тестов изменить статус на рабочий.
