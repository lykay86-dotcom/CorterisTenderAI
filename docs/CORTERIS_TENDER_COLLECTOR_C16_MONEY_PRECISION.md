# Техническая заметка FX1 — точность денежных значений

> Это дополнительное инфраструктурное улучшение, а не этап C16 основной
> дорожной карты Collector. Согласованный C16 — Stop-Factor Engine.

## Точность денежных значений

C16 устраняет преобразование ценовых границ через двоичный `float` внутри
поискового контура. НМЦК и границы фильтров теперь сравниваются как `Decimal`.

## Обратная совместимость

Публичные конструкторы продолжают принимать:

- `Decimal`;
- целые числа;
- `float` из существующего UI;
- десятичные строки из JSON и API.

После создания модели значение нормализуется в конечный неотрицательный
`Decimal`. Значения `NaN`, `Infinity` и отрицательные суммы отклоняются.

## Сохранённые профили

Новые версии профилей записывают `min_price` и `max_price` десятичными
строками. Это сохраняет точное значение при JSON round-trip. Старые профили,
в которых цена записана JSON-числом, читаются без миграции файла.

## Фильтрация и провайдеры

Локальный фильтр сравнивает `TenderMoney.amount` с границами напрямую, без
промежуточного `float`. Провайдеры получают тот же объект запроса и сохраняют
свои существующие форматы транспорта, необходимые конкретным площадкам.

## Гарантии C16

- большие значения не сливаются из-за ограничения точности IEEE-754;
- копейки сохраняются в профилях и checkpoint payload;
- существующие UI-формы и старые JSON-профили остаются совместимыми;
- денежные границы валидируются единообразно в модели, запросе и фильтре.
