# Безопасность и защита коммерческой версии

## Реалистичная цель
Полностью исключить декомпиляцию локальной Windows-программы невозможно. Цель — исключить простое извлечение исходников, существенно повысить стоимость анализа и не позволить локальной модификации разблокировать коммерческие функции.

## Слои защиты
1. Production-сборка через Nuitka.
2. Критические модули — Cython или Rust/PyO3.
3. Приватные ключи, платежи, entitlements и отзыв лицензий — на сервере.
4. Лицензионный токен подписывается сервером; в клиенте только публичный ключ.
5. Секреты пользователя — Windows Credential Manager/DPAPI.
6. EXE, DLL, updater и installer подписываются Authenticode.
7. Обновления имеют подписанный манифест и SHA-256.
8. UI не является единственной проверкой доступа.

## Security-тесты
- исходные .py не извлекаются из production-пакета;
- бинарник не содержит секретов;
- изменение тарифа/срока ломает подпись;
- перевод часов назад не продлевает подписку;
- подмена ответа сервера выявляется;
- прямой вызов сервиса не обходит entitlement;
- неподписанное обновление отклоняется.
