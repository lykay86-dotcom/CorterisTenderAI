# Collector C5 — Портал поставщиков Москвы

## Статус интеграции

Реализован нативный асинхронный адаптер к официальным сервисам Портала
поставщиков. Площадка документирует bearer-токен в заголовке `Authorization`
и отдельные методы поиска и получения карточки котировочной сессии.

Интеграция не считается подтверждённой на реальном аккаунте до проверки с
токеном ООО «КОРТЕРИС». Без токена провайдер не выполняет сетевой запрос и
возвращает состояние `not_configured`.

## Официальные методы

По документации Портала используются:

```text
https://api.zakupki.mos.ru/api/v2/auction/public/Search
https://api.zakupki.mos.ru/api/api/v2/auction/public/Get
```

Параметры передаются как JSON в query string. В официальном примере поиска
показаны фильтры `filter.name` и `startprice`.

Адреса можно переопределить без изменения исходников:

```text
CORTERIS_MOS_SEARCH_URL
CORTERIS_MOS_GET_URL
```

Это предусмотрено на случай изменения версии API.

## Настройка токена в PowerShell

Только для текущего окна PowerShell:

```powershell
$env:CORTERIS_MOS_API_KEY = "ВАШ_BEARER_ТОКЕН"
```

Токен не добавляется в Git и не печатается полностью в диагностике.

## Проверка подключения

```powershell
.\.venv\Scripts\python.exe scripts\check_mos_supplier_api.py
```

Поиск:

```powershell
.\.venv\Scripts\python.exe scripts\check_mos_supplier_api.py `
    --search "видеонаблюдение"
```

Карточка котировочной сессии:

```powershell
.\.venv\Scripts\python.exe scripts\check_mos_supplier_api.py `
    --id 9294080
```

Сохранение нормализованного результата для дальнейшей верификации полей:

```powershell
.\.venv\Scripts\python.exe scripts\check_mos_supplier_api.py `
    --search "СКУД" `
    --save .\diagnostics\mos_supplier_result.json
```

## Поддержанные данные

Адаптер нормализует:

- ID и номер котировочной сессии;
- название и описание;
- статус;
- заказчика и ИНН;
- регион;
- начальную цену через `Decimal`;
- дату публикации;
- срок окончания подачи предложений;
- ОКПД2;
- ссылки на карточку;
- документы и ссылки `FileStorage/Download`.

## Фильтры

В официальный запрос передаются только поля, подтверждённые документацией:

- `filter.name`;
- `startprice.start`;
- `startprice.end`;
- `startprice.isNotNull`.

Дополнительные поля могут быть переданы явно:

```python
TenderSearchQuery(
    keywords=("видеонаблюдение",),
    extra={
        "mos_api_payload": {
            "filter": {"status": [1, 2]}
        }
    },
)
```

Схема серверной пагинации пока не зафиксирована в доступном контракте,
поэтому текущая страница применяется локально после ответа. После первой
проверки реального токена нужно сохранить обезличенный ответ и подтвердить
поля пагинации.

## Fixtures

Файлы в `tests/fixtures/mos_supplier_*_documented_contract.json` построены
только из семантических полей, перечисленных официальной документацией. Они
не выданы за захваченные реальные ответы. Скрипт диагностики позволяет
получить нормализованный результат с реальным токеном для следующего этапа.

## Инкрементальный режим

Официальный cursor изменений в текущем контракте не подтверждён. Используется
checkpoint с семидневным перекрытием по дате публикации. История изменений
сохраняется Collector C3 после нормализации и дедупликации.

## Ограничения

1. Нужен bearer-токен, полученный по правилам Портала.
2. Реальный payload должен быть проверен на аккаунте Кортерис.
3. Серверная пагинация требует подтверждения фактическим ответом.
4. CAPTCHA, авторизация через браузер и защитные механизмы не обходятся.
5. Токен не записывается в crash-report и URL логов.
