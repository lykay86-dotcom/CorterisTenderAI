## Обязательный prerequisite RM-111 — воспроизводимый quality gate

Статус prerequisite: `DONE` (PR #22, merge `ebfdf01`). Обе обязательные
Windows-проверки на Python 3.12 и 3.13 прошли на PR и повторно на `main`;
branch protection требует PR, актуальную ветку и оба check context.

До реализации бизнес-логики AI Orchestrator в рамках RM-111 необходимо:

- изолировать offline-тесты от Windows Credential Manager и реальных API;
- исключить сетевой запрос при запуске тестов без явно разрешённого live-режима;
- добавить воспроизводимый Windows quality gate для Python 3.12 и 3.13;
- включить в обязательный gate полный pytest, Ruff check, Ruff format check и
  зафиксированный mypy-контур;
- сохранить C19 live-проверку отдельной ручной/разрешённой операцией;
- не изменять в этом пакете AI-бизнес-логику, C17 или C19.

Prerequisite считается выполненным только после прохождения полного baseline в
обычном пользовательском окружении, включая окружение с сохранёнными credentials,
без чтения этих credentials и без сетевых обращений из offline-тестов.

## RM-111 — implementation acceptance

Статус реализации: `DONE` (PR #24, merge `f246381`).

- создан единый `TenderAiOrchestrator` поверх существующего task-service;
- полный анализ больше не вызывает AI task-service или provider напрямую;
- текущий AI-результат явно передаётся в RM-107;
- status-to-warning policy и последняя exception boundary централизованы;
- production runtime создаёт один Orchestrator и один общий AI repository;
- legacy `structured_analysis` больше не является provider workflow и не
  рассчитывает собственные score/recommendation;
- UI показывает отдельную стадию, существующий HTML/JSON payload сохранён;
- миграция БД не требуется;
- целевой набор: `93 passed`; полный локальный регресс: `748 passed`;
- Ruff check, Ruff format check и mypy для 7 файлов проходят.

PR #24 слит в `main` 13 июля 2026 года. Quality Gate merge-коммита
`f246381` успешно прошёл на Python 3.12 и 3.13. RM-111 соответствует
Definition of Done; следующим активным этапом назначен RM-112.

## RM-112 — implementation acceptance

Статус реализации: `DONE` (PR #26, merge `1d559b5`).

- канонический persisted source — секция `ai` существующего `ConfigManager`;
- stable provider IDs: `disabled`, `openai`, `openai_compatible`;
- переиспользованы `DisabledProvider` и единый `OpenAICompatibleProvider`;
- default, unknown/corrupt config и ошибки credential storage дают безопасный
  `disabled` fallback без чтения keyring в disabled flow и без сетевого вызова;
- legacy display label не активирует provider, non-secret draft migration
  идемпотентна;
- выбранный provider внедряется bootstrap в существующий runtime с одним
  analyzer, Orchestrator и repository;
- переиспользована существующая ChatGPT/ИИ вкладка, secret не хранится в UI
  или config, сохранение не выполняет connection test;
- local/Ollama не реализован и остаётся объёмом RM-113;
- миграция БД не требуется;
- целевой набор: `62 passed`; полный локальный регресс: `784 passed` за 52,92 с;
- Ruff check/format, mypy для 9 файлов, secret scan, dependency audit и
  `git diff --check` проходят.

PR #26 слит в `main` 13 июля 2026 года коммитом `1d559b5`. Post-merge Quality
Gate успешен на Python 3.12 и 3.13. Следующим активным этапом назначен RM-113;
его application-код требует отдельного аудита.
