# Дорожная карта CorterisTenderAI

## Назначение

Этот документ является каноническим планом развития CorterisTenderAI.

Общая цель проекта:

**поиск → импорт → документация → анализ → оценка → смета → решение → подготовка заявки → контроль исполнения → финансы → управленческая аналитика.**

План рассчитан до **RM-200**, после которого фиксируется версия **CorterisTenderAI 2.0**.

## Правила ведения

1. Каждый пункт имеет постоянный идентификатор `RM-XXX`.
2. Номер GitHub Issue не заменяет номер дорожной карты.
3. Новая функциональность должна быть привязана к пункту дорожной карты.
4. Пункт не считается завершённым только потому, что код написан.
5. Для завершения необходимы тесты, проверка интерфейса и обновление документации.
6. Если функциональность уже существует, сначала проводится аудит, после чего пункт получает статус `PARTIAL` или `DONE`.
7. Изменения дорожной карты фиксируются в `docs/ROADMAP_HISTORY.md`.
8. Переход к следующему пункту разрешён только после выполнения критериев `Definition of Done`.

## Статусы

- `DONE` — завершено и проверено;
- `PARTIAL` — реализовано частично;
- `IN PROGRESS` — выполняется сейчас;
- `BLOCKED` — заблокировано внешней причиной;
- `PLANNED` — запланировано;
- `SUPERSEDED` — заменено другим решением.

## Текущее состояние

- `RM-001`–`RM-106`: завершены по рабочей дорожной карте, требуют периодической сверки с кодом;
- `RM-107`: текущий этап;
- `RM-108`–`RM-200`: запланированы.

- не участвовать;
- недостаточно данных для решения.
