C15 добавляет отдельную стадию между проверкой происхождения полей и
расчётом рейтинга:

```text
Verification

```

## Срок подачи

Для срока сохраняются:

- исходное значение источника;
- исходный часовой пояс;
- способ подтверждения часового пояса;
- нормализованное время UTC;
- время в часовом поясе пользователя;
- оставшееся количество секунд;
- признак завершённого срока.

Поддерживаются:

- timezone-aware `datetime` с явным UTC-offset;
- IANA timezone, когда база `zoneinfo` доступна;
- явные смещения `+03:00`, `UTC+3`, `GMT+03:00`;
- обозначения `UTC`, `GMT`, `MSK`, `МСК`.

Naive `datetime` без указанного источником часового пояса не считается
московским автоматически. Такой срок получает состояние «часовой пояс не
подтверждён» и требует повторной проверки.

Провайдер может передать метаданные:

```python
raw_metadata={
    "application_deadline_original": "15.07.2026 18:00",
    "application_deadline_timezone": "+03:00",
}
```

или через provenance:

```python
raw_metadata={
    "field_provenance": {
        "application_deadline": {
            "original_value": "15.07.2026 18:00 МСК",
            "source_timezone": "MSK",
        }
    }
}
```

## Политика повторной проверки

| Условие | Интервал |
|---|---:|
| До срока менее 6 часов | 30 минут |
| До срока менее 24 часов | 1 час |
| До срока менее 48 часов | 3 часа |
| Только агрегатор / публичная карточка | 1 час |
| Официальный сайт заказчика | 6 часов |
| Подтверждённый официальный источник | 24 часа |
| Нерешённый конфликт | немедленно |
| Неизвестный часовой пояс | немедленно |
| Источник сообщил об изменениях | немедленно |

Завершённая закупка отмечается отдельно и не ставится в очередь
автоматической перепроверки.

## Метаданные, ускоряющие перепроверку

Следующие флаги в `raw_metadata` назначают немедленную проверку:

- `official_source_unavailable`;
- `verification_required`;
- `has_changes`;
- `documents_changed`;
- `clarifications_changed`;
- `new_revision_available`.

## Хранение

Версия схемы Collector: `5`.

Таблица:

```text
collector_tender_freshness_state
```

Основные поля:

```text
status
last_verified_at
verification_due_at
is_stale
stale_reason
deadline_original
source_timezone
timezone_status
deadline_utc
user_timezone
deadline_user_local
seconds_remaining
recheck_interval_minutes
deadline_expired
```

При чтении реестра просроченная дата `verification_due_at` динамически
преобразуется в состояние `STALE`, даже если приложение не работало в момент
наступления срока повторной проверки.

## Планировщик

Планировщик продолжает использовать выбранный пользователем профиль и список
источников. Когда запись Collector достигает `verification_due_at`, запуск с
причиной `freshness_due` получает приоритет над обычным часовым или дневным
расписанием. Параллельный второй сбор не запускается.

## Интерфейс

В реестре добавлена колонка «Свежесть»:

- Актуально;
- Срок менее 48 ч;
- Требуется проверка;
- Подача завершена;
- Не подтверждено.

Tooltip показывает исходный срок, часовой пояс источника, UTC, локальное
время пользователя, последнюю и следующую проверку.
