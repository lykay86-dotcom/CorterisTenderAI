# Collector C8 — запуск, остановка и прогресс

## Назначение

C8 связывает асинхронное ядро Collector C2–C6 с основным интерфейсом
CorterisTenderAI. Пользователь выбирает сохранённый профиль, отмечает
включённые источники и запускает один фоновый сбор.

## Новый пункт интерфейса

```text
Тендеры → Запустить сборщик тендеров…
Ctrl+Shift+C
```

Окно показывает:

- выбранный профиль и его параметры;
- включённые источники;
- очередь и выполнение каждого источника;
- количество найденных записей и время ответа;
- ошибку конкретного источника;
- стадии нормализации, дедупликации и сохранения;
- новые, изменённые, дублирующиеся и сохранённые тендеры.

## Фоновая работа

`_CollectorRunWorker` запускается в `QThreadPool` и создаёт собственный
`asyncio` event loop. `CollectorRunSession` создаёт отдельный
`CollectorNetworkRuntime` для каждого запуска и закрывает его в `finally`.
GUI-поток не выполняет сетевые запросы.

## Остановка

Кнопка «Остановить» устанавливает потокобезопасный
`CollectorCancellationToken`. Активные async-запросы получают сигнал отмены.
Результаты источников, завершившихся до нажатия кнопки, не теряются:
они проходят нормализацию, дедупликацию и сохраняются в реестр. Запуск получает
статус `cancelled`.

## Прогресс

Ядро передаёт неизменяемые `CollectorProgressEvent`:

```text
preparing
provider_queued
provider_running
provider_completed
normalizing
deduplicating
saving
completed / cancelled / failed
```

Исключение внутри UI callback журналируется и не прерывает сбор.

## Сохранение

`CollectorService` сохраняет результаты через `CollectorStateRepository` в
существующий файл:

```text
<data>/tender_registry.sqlite3
```

Используются таблицы `tender_records`, `collector_runs`,
`collector_run_providers`, `collector_run_items`, история версий и изменений.
Повторный запуск не создаёт дубли.

## Коммерческие источники

`CollectorRunSession` запрашивает коммерческий каталог, но factory включает
только источники, разрешённые пользовательскими настройками. Непроверенный
адаптер возвращает честный статус `not_configured`; это не останавливает ЕИС
или Портал поставщиков.

## Обратная совместимость

Старый ручной поиск по профилям не изменён. Новые аргументы progress callback
необязательны. Старые fake engines и тесты продолжают работать, поскольку
`CollectorService` не передаёт новый аргумент, когда callback отсутствует.
