# Collector C19 — Verified Vertical Source

C19 вводит обязательный live-smoke шлюз для статуса источника `WORKING`.
Обычный health-check больше не является достаточным доказательством работы
вертикального источника.

## Обязательный маршрут

Один живой запуск должен последовательно подтвердить:

1. поиск;
2. получение карточки;
3. получение документов;
4. сохранение provenance;
5. field verification;
6. запись и чтение из БД;
7. отображение результата в UI;
8. полный анализ.

Результат каждого этапа содержит признак успеха, описание, количество
артефактов и длительность. История хранится в
`collector_vertical_source_verifications`. Fixture-тест, успешный health-check
или неполный маршрут не могут присвоить `WORKING`.

## Текущее фактическое состояние на 12 июля 2026 года

- `CORTERIS_MOS_API_KEY` не настроен: Портал поставщиков остаётся
  `NOT_CONFIGURED`;
- live health-check публичного интерфейса ЕИС завершился сетевым тайм-аутом
  после трёх попыток и примерно 63 секунд;
- обход CAPTCHA, HTTP 403 и других ограничений не выполнялся;
- ни одному источнику статус `WORKING` в рамках C19 не присвоен.

До успешного внешнего запуска UI показывает для доступного по health-check,
но не проверенного полного маршрута состояние `UNVERIFIED`.

## Условие завершения этапа

Нужен действующий токен официального API Портала поставщиков либо доступный и
разрешённый публичный канал ЕИС. После этого выполняется живой маршрут с
реальной закупкой и документами. Только сохранённый успешный результат всех
восьми этапов завершает C19 и переводит конкретный источник в `WORKING`.
